1 概要

1.1 目的とスコープ

このセクションは規定ではない。

この仕様、EPUB 3.1 は、デジタル出版物やドキュメントのための配信と交換形式の規格を定義する。EPUB® は、単一のファイル形式で配信するために体系化と意味論的に強化された Web コンテンツ（HTML5、CSS、SVG や他のリソースを含む）のパッケージングとエンコーディングの手段を定義している。

EPUB は、デジタルの本（電子書籍）の形式として広く採用され、この仕様は、複雑なレイアウト、リッチメディアと双方向性、グローバルなタイポグラフィの機能を含む、出版物が要求する幅広い範囲をサポートするために、形式の能力を大きく向上させている。EPUB 3.1 の期待は、本、雑誌、学術書、専門書や科学的な出版を含むコンテンツの幅広い範囲のために利用されるということである。

EPUB 3.1 は、事実上モジュール化されており、コア機能と規格の機能を定義する仕様群で構成されている。この仕様は、規格への主なエントリ ポイントを表すが、仕様に記載されている仕様はすべての規格に属する。主な概念と、これらの仕様を横断して定義された定義へのインデックスは、インデックスで提供される。

規定ではない [EPUB3Overview] は、規格への一般的な紹介を提供する。規格の以前のバージョンである EPUB 3.0.1 からの技術的な変更点の一覧は、規定ではない [EPUB3Changes] でも利用できる。

1.2 用語

製作者

EPUB 出版物の制作に対して責任がある人または組織。制作者は、コンテンツの制作者である必要はない。

コンテンツ提供者

流通や販売チャンル、または複数のリーディング システムへ直接的に EPUB 出版物を提供する出版社や著者、個人、他の情報源。

コア メディア タイプ

EPUB リーディング システムがサポートするメディア タイプ [RFC2046]。詳細については出版物リソースを参照されたい。

コア メディア タイプ リソース

コア メディア タイプであるため、フォールバックの提供が必要ではない出版物リソース（外部リソースを参照）。

初期設定のレンディション

container.xml [OCF31] ファイルの最初の rootfile 要素に記載しているレンディション。

EPUB コンテナ（またはコンテナ）

OCF ZIP コンテナ [OCF31] で定義されている EPUB 出版物用 ZIP ベースのパッケージングと配布の形式。

EPUB コンテンツ ドキュメント

EPUB コンテンツ ドキュメントの定義（HTML または SVG）のひとつに一致した出版物リソース。

EPUB コンテンツ ドキュメントはコア メディア タイプであるため、フォールバックを提供することなく EPUB 出版物に含まれていてもよい。

EPUB ナビゲーション ドキュメント

人間と機械が読み取り可能であるグローバルなナビゲーション情報を含む HTML コンテンツ ドキュメントの特化。EPUB ナビゲーション ドキュメントは、EPUB ナビゲーション ドキュメント [Packages31] で述べられた制約に準じる。

EPUB パッケージ（またはパッケージ）

パッケージ ドキュメントで定義されているように、EPUB 出版物のひとつのレンディションをレンダリングする相互関係にあるリソースのセットから構成されている論理的なドキュメントのエンティティ。

EPUB 出版物

この仕様に準拠したひとつ以上のレンディションのコレクションは、EPUB Container をパッケージ化している。

EPUB 出版物は典型的に単一の知的・芸術作品を表しているが、この仕様は、コンテンツの性質を制限しない。

EPUB リーディング システム (またはリーディング システム)

この仕様に準拠する方法で読者に提示するために EPUB 出版物を処理するシステム。

固定レイアウト ドキュメント

rendition:layout プロパティ [Packages31] で定義されているように、パッケージ ドキュメント内の pre-paginated を指定されている spine から直接参照される EPUB コンテンツ ドキュメント。

固定レイアウト ドキュメントをレンダリングするために使用する寸法は、固定レイアウト [ContentDocs31] で定義される。

異種リソース

コア メディア タイプではない出版物リソース。異種リソースは、異種リソースで定義されているフォールバックの要件に依存する。

HTML コンテンツ ドキュメント

HTML コンテンツ ドキュメント [ContentDocs31] で定義されている [HTML5] のプロファイルに一致している EPUB コンテンツ ドキュメント。

[HTML5] に定義された HTML かそれとも XHTML 構文を使用する HTML コンテンツ ドキュメント。

マニフェスト

EPUB 出版物の指定されたレンディションを構成するすべての出版物リソースのリスト。

詳細についてはマニフェスト [Packages31]を参照されたい。

メディア オーバーレイ ドキュメント

[MediaOverlays31] で定義された同期再生を提供するために録音済みの音声ナレーションを HTML コンテンツ ドキュメントと関連付ける XML ドキュメント

パッケージ ドキュメント

パッケージ ドキュメント [Packages31] で定義されているように、EPUB 出版物のひとつのレンディションを記述する出版物リソース。パッケージ ドキュメントは、レンディションに関するメタ情報を伝達し、リソースのマニフェストの提供と標準の読み取り順序を定義する。

出版物リソース

コンテンツまたは EPUB 出版物の少なくともひとつのレンディションのロジックと表示に関する命令を含んでいるリソース。このリソースがない場合には、EPUB 出版物は製作者の意図したとおりに表示されないことがある。出版物リソースの例としてはレンディションのパッケージ ドキュメント、EPUB コンテンツ ドキュメント、CSS スタイル シート、音声、動画、画像、埋め込みフォントとスクリプトが含まれている。

パッケージ ドキュメント自身を除いて、レンディションをレンダリングするために必要な 出版物リソースはレンディションのマニフェスト [Packages31] に記載され、（出版物リソースの配置で特に指定がない限り）EPUB コンテナファイルにバンドルする。

出版物リソースでないリソースの例としては、パッケージ ドキュメントの link [Packages31] の外で解決する外部へのハイパーリンクで識別されるものが含まれている。（例えば [HTML5] a 要素の href 属性）

リモート リソース

一般的に、オンラインである必要はないが、EPUB コンテナの外部に配置するリソース

リソース配置に関するメディア タイプの特定のルールは、出版物リソースの配置を参照されたい。

公開リソース

公開リソースは EPUB 出版物の任意のインスタンスが、別のインスタンスに対して、それらが同一であるのか、異なる版なのか、または無関係なのかどうかを判断するために比較できるようにする。

詳細については公開リソース [Packages31] を参照されたい。

レンディション

EPUB パッケージで述べられているように、EPUB 出版物コンテンツのひとつのレンダリング。

初期設定のレンディション

コンテナ - META-INF/container.xml [OCF3] ファイルの最初の rootfile 要素に記載しているレンディション。

スクリプト コンテンツ ドキュメント

スクリプトや HTML5 フォーム要素を持つ HTML コンテンツ ドキュメントが含まれる EPUB コンテンツ ドキュメント。

詳細についてはスクリプト [ContentDocs31] を参照されたい。

Spine

EPUB 出版物の指定されたレンディションの初期設定の読み上げ順序を表す出版物リソース、一般的 [Packages31]に EPUB コンテンツ ドキュメントの順序付きリスト。

詳細については spine [Packages31] を参照されたい。

SVG コンテンツ ドキュメント

SVG コンテンツ ドキュメント [ContentDocs31] で表現された制約に準拠した EPUB コンテンツ ドキュメント。

SVG ビューポート

SVG 画像を表示できるビューポート。

合成見開き

デバイスの画面上で同時に二つの隣接するページのレンダリング。

テキスト読み上げ（TTS）

人工的な人間の発音で EPUB 出版物のテキストコンテンツのレンダリングは合成音声を用いる。

トップレベル コンテンツ ドキュメント

直接かフォールバックチェーン [Packages31] を介して、spine から参照される EPUB コンテンツ ドキュメント。

一意識別子

unique-identifier [Packages31] 属性によって識別される一意識別子は EPUB 出版物のための主要な識別子である。一意識別子は、同じ EPUB 出版物のひとつ以上のレンディションによって共有が可能である。

コンテンツの大幅な見直し、要訳などは新しい一意識別子が必要である。

読者

EPUB リーディング システムを使用して EPUB 出版物を消費する個人。

ユーザーエージェント

一般的な HTML を消費する包括的にクライアントまたはアプリケーション（すなわち、Web ブラウザ、スクリーンリーダー）。

ビューポート

EPUB 出版物の内容が視覚的に読者に与えられる EPUB リーディング システムの領域。

1.3 表記上の規則

次の表記上の規則は、この仕様に用いられる：

markup

すべてのマークアップ（要素、属性、プロパティ）、コード（JavaScript、擬似コード）、機械処理可能な値（文字列、文字、メディアタイプ）とファイル名は、赤色の等幅フォントで示している。

markup

マークアップとコードの定義へのリンクは、赤色の下線の等幅フォントで示している。

http://www.idpf.org/

URI は紺色の等幅フォントで示している。

hyperlink

ハイパーリンクは、青色と下線で示している。

[reference]

規定と規定ではない参考は、角括弧で囲まれている。

Term

用語集で定義されている用語は、キャピタルケースで示している。

Term

用語の定義へのリンクは、点線の青い下線である。

規定の要素と属性、プロパティの定義は、青いボックスにある。

規定ではないマークアップ例は、ライトグレーのボックス内にある。

note

規定ではない注記は、"Note"の見出しがある緑色のボックス内にある。

caution

規定ではない警告は、"Caution"の見出しがある赤いボックス内にある。

1.4 適合性宣言

このドキュメント内のキーワード、must、must not、REQUIRED、SHALL、SHALL NOT、should、should not、RECOMMENDED、may、OPTIONAL は、[RFC2119] の記述に従って解釈される。

この仕様のすべてのセクションは規定である。但し"このセクションは規定ではない情報である"という規定ではない状態のラベルで識別される箇所を除く。セクションと付録への規定ではない状態の適用は、含まれる可能性のあるすべての子コンテンツおよびサブセクションに適用される。

この仕様のすべての例は規定ではない。

1.5 廃止予定と置き換えられた機能

この仕様と兄弟仕様で定義された機能は、それらが使われなくなった場合や/またはそれらが新しい機能によって置き換えられた場合、折にふれて削除される。時々、このような使われなくなった機能は、改定時で完全に廃止予定されるだろうが、より頻繁に段階的な廃止予定が起こるだろう。このような場合、機能は、廃止予定する前に、廃止予定かそれとも置き換えとしてマークされるだろう。これらのラベルの意味は、次のように定義されている。

機能が、廃止予定としてマークされた場合、次が当てはまる。

  • 機能は、仕様の将来のバージョンで削除されるであろう。機能は、製作者や EPUB リーディング システム開発者が削除する準備の時間を与えるために、廃止予定としてマークされる。

  • 製作者が、それらの EPUB 出版物に、その機能の使用を中止することが強く推奨される（recommended）。

  • リーディング システムは、それが正式に削除されるまで、定義されているとして、機能のサポートを続けなければならない（must）。

  • 検証ツールは、EPUB 出版物に遭遇したとき、廃止予定された機能の包含をする製作者にアラートしなければならない（must）。アラートは、エラーやウォーニングより重大性が低いことが推奨される（recommended）。

機能が、置き換えとしてマークされた場合、次が当てはまる。

  • 置き換えられる機能は、この仕様の将来のバージョンで削除されてもよいが（may）、削除の予定は確定していない。

  • 製作者は、この仕様の過去のバージョンとの互換性のために、置き換えの機能を包含してもよいが（may）、この仕様と適合させるために代わりの機能を使用しなければならない（must）。

  • リーディング システムは、仕様のこのバージョンと適合するコンテンツにある置き換えられる機能をサポートしてはならない（must not）。

  • 検証ツールは、後方互換性のために妥当であるとして、EPUB 出版物の置き換えられる機能の存在ついて、製作者へアラートをしてはならない（must not）。検証ツールは、置き換えられる機能が、定義に適合していないか、さもなければ使用法の要件を破る場合は、製作者へアラートをしなければならない（must）。

2 組織

2.1 仕様

EPUB 3.1 規格は、サブ仕様のファミリーを横断して定義されたコア機能と機能性を持つ性質のモジュールである。

この仕様は、ファミリーの中で最上位の仕様を表している。これは、規格のプロダクトである EPUB 出版物と EPUB 出版物を消費し、読者にその内容を提示するアプリケーションである EPUB リーディング システムのための適合性要件が含まれている（EPUB 出版物の適合性と EPUB リーディング システムの適合性のそれぞれを見よ）。

完全な EPUB 3.1 の他の仕様は、次の通りである。

  • EPUB Packages 3.1 [Packages31] は、コンテンツの各レンディションの要件を定義している。

  • EPUB Content Documents 3.1 [ContentDocs31] は EPUB 出版物のコンテキストで使用するための XHTML、SVG と CSS のプロファイルを定義する。

  • EPUB Media Overlays 3.1 [MediaOverlays31] は、形式とテキストと音声の同期のための処理モデルを定義する。

  • EPUB Open Container Format (OCF) 3.1 [OCF31] は、関連するリソースの集合を単一のファイル（ZIP）の EPUB コンテナにカプセル化するためのファイル形式と処理モデルを定義する。

  • Alternate Style Tags [AltStyleTags] は、マイクロフォーマットを使用して代替のスタイルシートをタグ付けするためのパターンを定義している。

これらの仕様は、EPUB 3.1 に関連しているように広く認められている正式なリストであり、規格の一部として関連する機能を含んでいるが、新しい機能もまた拡張仕様の開発を通じて定期的に追加される。

OPEN ISSUE

最近完了したモジュールと進行中の作業の数は、コア規格の統合/一体化のために考慮中である。2015年12月15日の電話会議におけるワーキンググループのほぼ一致した意見としては、実績のある実装の不足のせいで、この時点ではこれらのモジュールを考慮しなかった。しかしながら、これらの包含は、将来のドラフトで再検討されるかもしれない。モジュールのリストについては、トラッカーの issue 635 を見よ。

規格のためにコアのリビジョンの外側で定義されている特徴と機能は、この仕様で正式に認められていないが、とはいえ、参照仕様として認識された時点で、製作者と開発者によって使用するために利用可能である。

note

現在および最近完了した活動のリストは、http://www.idpf.org/ongoing でメンテナンスされる。

2.2 ロードマップ

このセクションは規定ではない。

OPEN ISSUE

このロードマップは、以前は、概要にあったが、多くのコンテクストを与えるために新しいトップレベルの仕様に移された。手助けにならないと見なされれば、概要へ戻されるかもしれない。

ロードマップは、この時点で更新される。改定の結果が安定したら、後のドラフトで更新される。

EPUB 規格で最も重要なのは、EPUB 出版物である。このセクションでは、EPUB 出版物の作成に通じるさまざまな側面について簡単に紹介する。詳細については、参照仕様から入手可能であり、EPUB 3.1 の機能のより一般的な導入は、参考情報で提供される [EPUB3Overview]。

EPUB 出版物は、その最も基本的なレベルで、確実にまた予想通りに読者へそのコンテンツを表示するために、EPUB リーディング システムに取り込まれることができるリソースがバンドルされたコレクションである。 それぞれのEPUB 出版物は、レンディションと呼ばれるひとつまたはコンテンツのレンダリングから構成される。

EPUB Container のリソースの一部は、レンディションの発見と処理を容易にするものであり、他のリソースはもとの出版物の内容を組むために用いられる。後者の EPUB コンテンツ ドキュメントは、[ContentDocs31] で定義されている。

一般的に、EPUB 出版物のリソースは、ファイル拡張子 .epub の ZIP ベースのアーカイブとして、配布のためにひとまとめにされる。ZIP アーカイブ準拠なので、EPUB 出版物は多くのソフトウェアプログラムによって解凍でき、生産と消費の両方を単純にすることができる。コンテナフォーマットは、[OCF31] で定義されている。

コンテナフォーマットは、zip 圧縮されたコンテンツが EPUB 出版物であることを決定する手段（mimetype ファイル）だけでなく、有益なリソースの普遍的な名前のディレクトリも提供する（/META-INF）。これらの間のキーになるのは、（それら パッケージ ドキュメントの）EPUB 出版物の利用可能なレンディションのルートファイルをリーディング システムに指示する container.xml である。

パッケージ ドキュメントは、それ自身が特定のレンディションのための一元化された情報リソースのようなもので、レンディションによって表現される特定の作品についてのメタデータを格納し、リソースの完全なリストを提供し、初期設定の読み上げ順を定義する。パッケージ ドキュメントは、パッケージ ドキュメント [Packages31] で定義されている。

メディア オーバーレイ ドキュメントは、特定のレンディションのためにテキストと音声の再生を同期させる方法を定義する。オーバーレイ ドキュメントは、[MediaOverlays31] で定義されている。

以下の例は、"Hello World" EPUB 出版物に含まれている最小限の単一レンディションのリソースを示している：

mimetype
META-INF/container.xml
Content/HelloWorld.opf
Content/HelloWorld.xhtml

概念的には単純であるが、EPUB 出版物は、この例で示されるように、ZIP パッケージ内の HTML ページや依存資産の単純な集積以上のものである。このドキュメントの次のセクションは、読書体験を高めるのに提供する EPUB 出版物の主要な特徴と機能性についてより詳細に掘り下げている。

3 適合性

3.1 EPUB 出版物

EPUB 出版物は、次の条件をすべて満たさなければならない（must）：

パッケージ

[Packages31] に定義された EPUB パッケージの要件に準拠したコンテンツのレンディションを少なくともひとつ包含しなければならない（must）。

出版物リソース

そこに含まれるすべての出版物リソースは、コア メディア タイプと異種リソースの制約に従い、出版物リソースの配置ごとに配置しなければならない（must）。

コンテナ

[OCF31] で定義されているように、それは EPUB コンテナにパッケージ化しなければならない（must）。

OPEN ISSUE

ワーキンググループは、現在、標準化された EPUB 出版物の代替としてパッケージされていないマニフェステイションとブラウザベースのリーディング システムによるコンテンツの容易な横断を検討している。このマニフェステイションは、Open Container Format によって置き換えられないだろうし、二つの間を往復することができることを期待される。詳細については、サーバー/ブラウザの扱いやすそうなマニフェステイションドキュメントを参照されたい。

3.2 EPUB リーディング システム

EPUB リーディング システムは次の条件をすべて満たさなければならない（must）：

EPUB 3.1 の処理

[OCF31] で規定されているように EPUB コンテナを処理しなければならない（must）。

[Packages31] で定義されているように、EPUB パッケージを処理しなければならない（must）。

同じ一意識別子を持つ二つの異なる EPUB パッケージを検出した場合、壊滅的に機能しなくなってはならない（must not）。

このセクションの条件付き動作として指定されていない限り、すべてのコア メディア タイプ リソースをサポートしなければならない（must）。

異種リソースタイプの任意のセットをサポートしてもよい（may）。そして異種リソースで定義されているように、サポートされていない異種リソースのためのフォールバックを処理しなければならない（must）。

HTML コンテンツ ドキュメント - リーディング システムの適合性 [ContentDocs31] で定義されている HTML コンテンツ ドキュメントを処理しなければならない（must）。

SVG コンテンツ ドキュメント - リーディング システムの適合性 [ContentDocs31] で定義されている SVG コンテンツ ドキュメントを処理しなければならない（must）。

ビューポートがある場合、CSS スタイル シート - リーディング システムの適合性 [ContentDocs31] で定義されている HTML コンテンツ ドキュメントの視覚的なレンダリングをサポートしなければならない（must）。

ラスターイメージをレンダリングする機能がある場合、ラスター画像のコア メディア タイプをサポートしなければならない（must）。

ベクターイメージをレンダリングする機能がある場合、ベクター画像のコア メディア タイプをサポートしなければならない（must）。

録音済みの音声をレンダリングする機能がある場合、MP3 音声のコア メディア タイプをサポートしなければならない（must）。また MP4 音声のコア メディア タイプとメディア オーバーレイ [MediaOverlays31] をサポートするべきである（should）。

テキストの読み上げ（TTS）のレンダリングをサポートしている場合、発音辞書 [ContentDocs31]、EPUB CSS Profile [ContentDocs31] の CSS3 スピーチ機能と HTML コンテンツ ドキュメントの SSML 属性 [ContentDocs31] をサポートするべきである（should）。

note

リーディング システムは [H.264] と [VP8] ビデオコーデックの少なくともひとつをサポートすることが望ましいが、これは適合要件ではなく、リーディング システムは全ての動画コーデックをサポートできない。どちらか（または潜在的に、両方）の形式のビデオを含めたり、実装するために選択を行う際に、製作者やリーディング システムの開発者は、採用の幅、ビデオの再生品質、および技術の使用のロイヤリティの要件のような要因を考慮することが必要である。

下位互換性

[OPF2]、[OPS2] と [OCF2] で定義されている EPUB バージョン「2.0」出版物を処理するべきである（should）。

[Publications301] と [ContentDocs301]、[OCF301] で定義されているように、EPUB バージョン「3.0」出版物を処理するべきである（should）。

パッケージ ドキュメントの version 属性がバージョン「3.1」未満を指定されていたり、version 属性が削除されているどんな EPUB 出版物の指定されたレンディションでも処理しようと試みなければならない（must）。

前方互換性

Package Document の version 属性がバージョン「3.1」以上を指定されているどんな EPUB 出版物の指定されたレンディションでも処理しようと試みるべきである（should）。

XML の処理

適合性を検証しない処理 [XML] でなければならない（must）。

[XMLNS] に定義されているような、適合性の処理をしなければならない（must）。

xml-stylesheet 処理命令の [ASSOCSS] をサポートしなければならず（must）、追加の処理命令をサポートしてもよい（may）。

[XML Base] に定義されているような、適合性の適用をしなければならない（must）。

note

適合したリーディング システムは単一の専用プログラムまたはデバイスである必要はなく、分散システムとして存在してもよい。

4 出版物リソース

4.1 コア メディア タイプ

4.1.1 まえがき

一般的に、EPUB 出版物の各レンディションは、多くの出版物リソースから構成されている。それらのリソースは、二つのカテゴリーに分類される。それらは EPUB リーディング システムのサポートが必要なもの（コア メディア タイプ リソース）と、そうでないものである（異種リソース）。

製作者は、EPUB 出版物を構成するためのリソースに両方のタイプを使用するのは自由だが、いくつかのリーディング システムは、使用する外部リソースをレンダリングしないかもしれないことを知っておく必要がある。

EPUB 出版物は、全ての適合したリーディング システムで完全に消費するために設計されており、従って、フォールバックのシステムは、外部リソースの使用が、コンテンツを消費する読者の能力に影響を与えないことをを担保する必要性がある。このセクションは、消費の要件を満たすために使用することできるリーディング システムを横断してサポートされ、フォールバック メカニズムが識別するコア メディア タイプのセットを提示している。

4.1.3 異種リソース

異種リソースは、spine itemref 要素 [Packages31] から参照されず、また、（例えば、[HTML5] 埋め込みコンテンツや [SVG] image と foreignObject 要素を介して）EPUB コンテンツ ドキュメントにネイティブ形式で直接レンダリングされていないとき、フォールバックなしにレンディションに包含されてもよい（may）。

異種リソースが、spine に包含されるか、または、EPUB コンテンツ ドキュメントにネイティブ形式で直接レンダリングされるとき、コア メディア タイプのフォールバックは、包含されなければならない（must）。フォールバックは、次の二つのいずれかの形式を取る。

  • 固有のフォールバック メカニズムは、（例えば、ひとつ以上のメディア タイプを提供したり、メディア タイプをレンダリングできないときに代替の埋め込まれたメッセージを表示する能力のように）フォーマットに基いて提供される。それらの要素が提供する固有のフォールバック能力については [HTML5] と [SVG] 仕様を参照されたい。

  • マニフェスト フォールバック [Packages31] は、コア メディア タイプ リソースへのフォールバック チェインを作るパッケージ ドキュメントの機能である。それらは、spine [Packages31] 内と（例えば、[HTML5] img 要素など）固有のフォールバック能力が利用できないとき、異種リソースのフォールバックを作るために使用される。

スクリプトへの入力としての機能する異種リソースの場合、（例えば、[HTML5] canvas 要素内の描画に使用したり、HTML または SVG DOM 内に入れられるなど）スクリプトは、それがコア メディア タイプとして提示されるようデータを使用するべきである（should）。さもなければ、（例えば、[HTML5] 埋め込みコンテンツ要素にデータを書き込む場合、固有のフォールバックの代替を包含するなど）スクリプトは、コア メディア タイプ フォールバックを包含しなければならない（must）。

4.2 リソース配置

すべての出版物リソースは、次の例外を除いてEPUB コンテナに配置しなければならない（must）。

  • 音声リソースは、コンテナまたはリモートに配置されてもよい（may）。

  • 動画リソースは、コンテナまたはリモートに配置されてもよい（may）。

OPEN ISSUE

ワーキンググループは、現在、コンテナ内のリソースを、読者によるダウンロードが可能なようにマークできるだけでなく、例えばフォントやデータのセットなどの他のリソースが、コンテナの外でホスト可能かかどうかを検討している。詳細は、外部リソース ドキュメントを参照されたい。

製作者は接続状態に関わらずプレゼンテーション全体にユーザーのアクセスを許可するため、コンテナ内に音声や動画リソースを配置するのを選択するべきである（should）。

note

指定されたリソースがコア メディア タイプ リソースまたは異種リソースであるかどうか関係なく出版物リソースの配置については、上記のルールが適用される。

note

EPUB 出版物でリモート リソースの取り込みは、マニフェスト item [Packages31] 要素の remote-resources プロパティ [PackageVocab] を介して指示される。

4.3 XML 適合性

XML ベースのメディアタイプである全ての出版物リソースは、次の制約を満たさなければならない（must）：

  • それはドキュメントの適合性 [XMLNS] で定義されている XML1.0 に準拠したドキュメントでなければならない（must）。

  • 拡張識別子は文書型宣言 [XML] に現れてはならない（must not）。

  • それは XInclude [XInclude] を利用してはならない（must not）。

  • それは UTF-8 または UTF-16 [Unicode] でエンコードしなければならない（must）。

特定の出版物リソースが、コア メディア タイプ リソースまたは異種リソースであるか否かを問わず上記の制約はあてはまる。
